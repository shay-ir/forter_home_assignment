In this excercise you'll need to fix some bugs of lz-string library (http://pieroxy.net/blog/pages/lz-string/index.html)
You'll find its code attached as 'lz-string.js' file right here.
The tools you can use are anything you wish. You'll also find here a debug function that works in all browsers,
and can be called from the lz-stirng.js code, as long as you use this page to debug the code.

So, The mission is to make this page work in Internet Explorer 6. why? because we're evil :)
You can get a working environment by downloading a VM from http://dev.modern.ie/tools/vms/
You can use firebug lite to add some debug capabilities to IE 6, but I don't think it will do you any good (https://getfirebug.com/firebuglite)
Let's get to the code. If I run the following code in any decent browser:

var output = LZString.compressToEncodedURIComponent("Let's put another shrimp on the barbie!");
html_log(output);

I'll get: DIUwLg5AzgBADgVzDAhgOwPZgBYgE4xTZ4CWAtnDBmjDiDAEYp4MkgCEQA
But the same thing over IE6 will return: K418ZWkA

Your mission, if you wish to accept it, is to fix the lz-string code so that it will yield the same outputs when running in IE 6 as any other browser.
